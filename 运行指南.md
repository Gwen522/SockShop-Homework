# 运行指南（给要跑代码的同学）

> 这份是给**在别的电脑上运行/演示本项目**的同学看的。Mac 和 Windows 都涵盖了，照着做即可。
> 如果只是展示讲 PPT、不现场跑代码，可以跳过本文，直接用 `results/` 里的图表。

---

## 总览：要跑起来，需要三样东西

1. **代码** —— 从 GitHub 下载本项目
2. **环境** —— Docker、Python、Chrome、JMeter（下面有详细安装说明）
3. **按顺序执行命令** —— 起网站 → 跑测试

---

## 第一步：下载代码

任选一种：

- **方式 A（推荐，简单）**：打开仓库 https://github.com/Gwen522/SockShop-Homework
  → 点绿色的 **Code** 按钮 → **Download ZIP** → 解压。
- **方式 B（会用 git 的话）**：
  ```
  git clone https://github.com/Gwen522/SockShop-Homework.git
  ```

下载后，进入项目文件夹（下文统一叫「项目目录」）。

---

## 第二步：装环境

### 1) Docker（两个系统都要）
- 装 **Docker Desktop**：https://www.docker.com/products/docker-desktop/
- 装完打开它，等到图标不再转圈（表示后台服务起来了）。
- 不需要注册/登录账号，跳过即可。

### 2) Python（功能测试要用）
- Mac 一般自带；Windows 去 https://www.python.org/downloads/ 下载安装，
  **安装时务必勾选 "Add Python to PATH"**。
- 装完在项目目录里装依赖：
  - Mac： `pip3 install -r requirements.txt`
  - Windows： `pip install -r requirements.txt`

### 3) Chrome 浏览器（功能测试要用）
- 装最新版 Chrome 即可，驱动会自动下载，不用手动配。

### 4) JMeter（性能测试要用）
- 需要先装 **Java 11 以上**（https://adoptium.net/ 下载 Temurin 即可）。
- JMeter 本体：https://jmeter.apache.org/download_jmeter.cgi 下载 `apache-jmeter-5.6.3.zip`，解压备用。
  （注：仓库里没带 JMeter 本体，因为文件太大。）

> 只演示功能测试（Selenium）的话，JMeter 和 Java 可以先不装。

---

## 第三步：启动 SockShop 网站

在**项目目录**里打开终端（Windows 用 PowerShell 或 CMD），执行：

```
docker compose up -d
```

第一次会下载镜像，**可能要等几分钟**。完成后浏览器打开 http://localhost:8079 ，
能看到卖袜子的网站就说明成功了。

> 如果镜像下载很慢或失败，是网络问题，可以给 Docker 配国内镜像源（搜"Docker 配置镜像加速"）。

---

## 第四步：跑测试

### A. 功能测试（Selenium）—— 两个系统通用

进入项目目录执行：

- Mac： `python3 selenium/test_sockshop.py`
- Windows： `python selenium/test_sockshop.py`

会自动跑 8 个用例并打印结果。加 `--no-headless` 可以看到浏览器自己操作：

```
python selenium/test_sockshop.py --no-headless
```

### B. 性能测试（JMeter）

> 把下面命令里的 `<JMeter路径>` 换成你解压 JMeter 后 `bin` 目录里 jmeter 的位置。

- **Mac / Linux**：
  ```
  <JMeter路径>/bin/jmeter -n -t jmeter/load_gradient.jmx -l result.jtl -Jthreads=100 -Jloops=20 -Jramp=5
  ```
- **Windows**：
  ```
  <JMeter路径>\bin\jmeter.bat -n -t jmeter\load_gradient.jmx -l result.jtl -Jthreads=100 -Jloops=20 -Jramp=5
  ```

跑完终端会打印一行 summary，其中 `Err` 那列是错误率（0 表示全部成功）。

## 现场演示的最快方式

如果是 **Mac**，项目里有个一键脚本，进项目目录直接：

```
zsh demo.sh
```

会弹菜单，输 `1`/`2`/`3` 选演示内容（`3` 是直接看现成结果，最快最稳）。

**Windows 没有 zsh，跑不了这个脚本**，请直接用上面「第四步」的命令，或者干脆打开
`results/` 文件夹里的图表 + `jmeter/html_report/index.html` 给大家看，一样达到展示效果。

---

## 常见问题

- **网站打不开 / 测试连不上**：先确认 Docker Desktop 开着、`docker compose up -d` 成功、
  浏览器能打开 http://localhost:8079 。
- **pip 装不上**：换国内源，命令后面加 `-i https://pypi.tuna.tsinghua.edu.cn/simple`。
- **Selenium 报浏览器相关错误**：确认装了 Chrome；或者改用 Safari（仅 Mac，需在 Safari 开发菜单开启"允许远程自动化"）。
- **不想装 JMeter**：那就只演示功能测试，性能部分直接看报告里的图表和数据。
